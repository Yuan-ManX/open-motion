## 概览

OpenMotion 是一个动效运行时——不是一个带预览窗口的设计工具，而是一片动效原生存活的执行表面。你不再面对一条只能凝视的时间轴，而是编程一个运行时：与动效 Agent 对话，它在动效真正运行的地方——HTML 与视频——构建、调优、交付。每一个模板在被选中的瞬间即已存活；每一个组件都可以用自然语言重塑；你打磨出的每一段动效，都能被打包成 **skill**，交给任意 AI 复用。每一次变更都被检查点守护，每一个实体都可被检索，每一个计划都可审视、可取消。

动效在这里是活的、可编程的、事务化的产物——而非静态文件。


## 为什么是 OpenMotion

传统动效工具分裂为两个阵营：网页画布工具止步于简单转场，时间轴厚重的动效图形套件则困在导出管线之后。OpenMotion 以原创架构闭合这条裂缝：一张活的、Web 原生画布——动效在哪里运行，就在哪里交付——配以从第一性原理自研的完整动效图形引擎：动态模糊、空对象、路径修剪、重复器、运动拖尾、时间重映射、图层效果。全部浏览器原生，全部 AI 原生。

让它成为**运行时**——而非工具——的是底层基础设施：检查点回滚语义让每一次变更都可逆；FTS5 全文检索让每一个项目、组件、消息、记忆与 skill 都可被查询；先规划后执行的编排让复杂多步操作可审视、可取消；MCP 原生表面让运行时可被任意外部 Agent 编程。设计工具产出文件。运行时托管活的、可寻址的、事务化的状态。

我们围绕**闭环**优化——意图、预览、精修、复用——这是真实产品工作真正需要的那条循环。我们把一段动效视为**可传递的设计知识单元**，它在人与 AI 之间流动。

- **动效即代码，而非工程文件。** 无导入，无导出管线。动效*本身*就是页面。
- **对话即光标。** 你描述意图，Agent 编辑曲线、节奏与图层。
- **事务化设计。** 每一次 spec 变更都被检查点守护；回滚为 O(1)。运行时永不丢失你的工作。
- **端到端可检索。** FTS5 检索覆盖项目、组件、消息、记忆、skill、生成式 skill 与 recipe——一次查询，全部实体。
- **先规划后执行。** 复杂请求分解为类型化、可审视的动作，带实时进度与中途取消。
- **一画布、一引擎、一 Agent。** 活的网页预览与专业动效图形引擎融为一体——同一表面，同一时间轴，同一个 Agent。
- **Skill 是通货。** 一段优秀的动效成为任意 AI Agent 可调用的复用原语。
- **MCP 原生。** 外部 Agent 通过 Model Context Protocol 编程运行时——动效作为头等 Agent 表面。


## 功能

### 一句话生成精致动效
通过对话产出专业级动画。描述感觉，Agent 塑造帧。需要精确时随时切换到关键帧直接控制——对话式编辑与手动时间轴并存。

### 专业动效图形，浏览器原生
完整的动效图形引擎，无需导出管线。超过 340 个工具、23 个原生系统，每一项原语都可被 Agent 调用：

- **图层与合成** — 动态模糊、空对象、路径修剪、重复器、运动拖尾、时间重映射、图层效果、高级混合、预合成、Alpha 与转移模式、混合组。
- **3D 摄像机与光学** — 摄像机与灯光、景深、镜头畸变、色差、暗角、相机抖动、光流、变形光晕。
- **调色** — 色阶、曲线、色彩平衡、色相/饱和度、自然饱和度、曝光、可选颜色——全部为原生 SVG 滤镜链。
- **路径与形状** — 布尔运算、多路径修剪/合并/偏移、收缩与膨胀、扭转、锯齿。
- **时间与节奏** — 时间位移、回声、带过渡的序列、时间反转、定格帧、抽帧、时间重映射变速。
- **文字动画** — 范围选择器、文字抖动器、路径文字、逐字符变换、文字动画器。
- **效果与滤镜** — 模糊（高斯/方向/径向）、锐化、波形扭曲、涟漪、膨胀、发光、马赛克、查找边缘、镜头光晕、四色渐变。
- **表达式** — 活的表达式引擎，原生支持循环、序列、指数缩放、平滑/抖动关键帧、音频关键帧。
- **绘画、克隆与遮罩** — 画笔笔触、克隆图章、画笔设置、遮罩系统与轨道遮罩。
- **数据与音频** — 数据绑定、CSV/JSON 数据源、数据驱动图表、音频驱动器、音频反应关键帧。
- **木偶与网格** — 基于钉点的骨骼变形与网格扭曲。
- **粒子与渐变** — 程序化粒子系统、渐变填充与描边（线性/径向/锥形）。
- **跟踪与稳定** — 点/摄像机跟踪、Warp 稳定器、运动路径编辑、自动朝向。
- **Rotoscoping 与抠像** — Roto Brush、Refine Edge、Color Key、Difference Matte、Spill Suppression、Matte Choker、Inner/Outer Key。
- **转场** — Block Dissolve、Card Wipe、Gradient/Iris/Linear/Radial Wipe、Venetian Blinds、CC Jaws。
- **模拟与生成器** — Ball Action、Bubbles、Rainfall、Snowfall、Star Burst、Cell Pattern、Audio Spectrum、Radio Waves。
- **Stylize** — Cartoon、Brush Strokes、Oil Paint、Watercolor、Emboss、Motion Tile、Scatter、Threshold。

### AI 原生动效智能
六套原创系统，把动效变为可查询、可杂交、可叙事、可自愈的介质——每一套都可被 Agent 作为工具调用，每一套都在 Intel 面板中呈现。

- **变体引擎（Variation Engine）。** 沿正交轴生成 N 个变体——缓动、时长、强度、方向、起点、错峰——一次往返探索整个设计空间。
- **动效 DNA（Motion DNA）。** 将任意组件分解为基因指纹：缓动家族、时序画像、变换签名、触发语义、强度、合成签名。两个组件可按 DNA 相似度（0..1）比较。
- **风格迁移（Style Transfer）。** 将源组件的缓动、时序、强度施加于目标——保留结构，采纳源的运动质感。
- **动效评审（Motion Critique）。** 四维结构化分析——可达性、性能、美学、一致性。返回带严重度标签的评分报告与建议。
- **动效叙事（Motion Storytelling）。** 叙事到动效的合成引擎。七种叙事意图分解为五幕弗赖塔格金字塔——每一幕拥有独立缓动、强度、情感色调与变换提示。
- **动效谱系（Motion Lineage）。** 基于 DAG 的组件谱系追踪。记录跨操作的父子关系，计算代际深度，回答祖先/后代/兄弟查询。
- **动效合成（Motion Synthesis）。** DNA 杂交，四种遗传策略：blend、dominant、crossover、mutation。每条合成 DNA 携带性状归因。按种子可复现。
- **动效自愈（Motion Auto-Fix）。** 一键可达性修复。检测前庭、光敏、降低动效、认知四类问题，并施加针对性修复——封顶位移与旋转、把闪烁拉长到 3Hz 阈值以下、封顶无限循环、设置 fillMode、错峰并发动画、归一化时序层级、统一缓动家族。返回修复前后分数与逐项审计轨迹。

### 动效作为可迁移单元
- **组件级精细调优。** 每一段动效由相互独立的组件构成。调一个，不动其余。
- **Skill 管线。** 把任意动效打包为自包含、可被 AI 调用的 skill——同一原语跨项目、跨团队、跨 Agent 流转。
- **多格式导出。** HTML/CSS、React、JSON、MP4、GIF、WebM——浏览器里运行的那个产物，就是你交付的那个产物。
- **MCP 原生。** 外部 Agent 通过 Model Context Protocol 编程运行时——动效作为头等 Agent 表面。

### 天生为 Agent 设计
OpenMotion 从一开始就为被 AI Agent 驱动而构建，而非仅供人类操作。Agent 不仅编辑关键帧——它运行变体引擎、提取动效 DNA、跨组件迁移风格、评审结构质量、生成叙事序列、追踪谱系、从多源合成杂交动效、一键自愈可达性违规——全部通过对话意图完成。

## 架构

OpenMotion 围绕五大奠基支柱组织——四大产品表面，加上让它们事务化、可检索、可编程的运行时基础设施：

| 支柱 | 职责 |
| --- | --- |
| **模板库** | 精选、持续增长的顶尖动效模板集合，开箱即活。 |
| **动效 Agent** | 对话式核心——把意图翻译为精确的参数与关键帧修改。 |
| **实时运行时** | 在真实网页与视频帧中运行动效；同一表面即交付表面。 |
| **Skill 管线** | 把任意动效打包为自包含、可被 AI 调用、可复用的单元。 |
| **运行时基础设施** | 检查点回滚、FTS5 全文检索、先规划后执行编排、MCP 原生 Agent 表面——让动效事务化、可检索、可编程的基座。 |

它们共同构成闭环：*选择 → 运行 → 精修 → 交付 → 复用*——每一步都被检查点守护、可被检索、可被任一 AI Agent 寻址。
