## 概览

OpenMotion 是一个动效运行时——不是一个带预览窗口的设计工具，而是一片动效原生存活的执行表面。你不再面对一条只能凝视的时间轴，而是编程一个运行时：与动效 Agent 对话，它在动效真正运行的地方——HTML 与视频——构建、调优、交付。每一个模板在被选中的瞬间即已存活；每一个组件都可以用自然语言重塑；你打磨出的每一段动效，都能被打包成 **skill**，交给任意 AI 复用。每一次变更都被检查点守护，每一个实体都可被检索，每一个计划都可审视、可取消。

动效在这里是活的、可编程的、事务化的产物——而非静态文件。


## 为什么是 OpenMotion

传统动效工具分裂为两个阵营：网页画布工具止步于简单转场，时间轴厚重的动效图形套件则困在导出管线之后。OpenMotion 以原创架构闭合这条裂缝：一张活的、Web 原生画布——动效在哪里运行，就在哪里交付——配以从第一性原理自研的完整动效图形引擎：动态模糊、空对象、路径修剪、重复器、运动拖尾、时间重映射、图层效果。全部浏览器原生，全部 AI 原生。

让它成为**运行时**——而非工具——的是底层基础设施：检查点回滚语义让每一次变更都可逆；FTS5 全文检索让每一个项目、组件、消息、记忆与 skill 都可被查询；先规划后执行的编排让复杂多步操作可审视、可取消；MCP 原生表面让运行时可被任意外部 Agent 编程。设计工具产出文件。运行时托管活的、可寻址的、事务化的状态。

我们围绕**闭环**优化——意图、预览、精修、复用——这是真实产品工作真正需要的那条循环。我们把一段动效视为**可传递的设计知识单元**，它在人与 AI 之间流动。

- **动效即代码，而非工程文件。** 无导入，无导出管线。动效*本身*就是页面。
- **对话即光标。** 你描述意图，Agent 编辑曲线、节奏与图层。
- **事务化设计。** 每一次 spec 变更都被检查点守护；回滚为 O(1)。运行时永不丢失你的工作。
- **端到端可检索。** FTS5 检索覆盖项目、组件、消息、记忆、skill、生成式 skill 与 recipe——一次查询，全部实体。
- **先规划后执行。** 复杂请求分解为类型化、可审视的动作，带实时进度与中途取消。
- **一画布、一引擎、一 Agent。** 活的网页预览与专业动效图形引擎融为一体——同一表面，同一时间轴，同一个 Agent。
- **Skill 是通货。** 一段优秀的动效成为任意 AI Agent 可调用的复用原语。
- **MCP 原生。** 外部 Agent 通过 Model Context Protocol 编程运行时——动效作为头等 Agent 表面。


## 功能

### 一句话生成精致动效
通过对话与 OpenMotion 动效 Agent 协作，产出专业级动画。无论是单个重复动作，还是完整的多场景叙事——描述感觉，Agent 塑造帧。

### 完全掌控每一个关键帧
当你需要精确，时间轴归你所有。精确设定每一帧与每一属性。对话式编辑与关键帧控制并存——任意时刻自由切换。

### 广阔的创造力，精确的控制
为作品注入清晰与表现力。动画化加载状态、悬停交互、Logo 揭示、转场——时间轴就是上限。

- **预制风格原语。** 精选风格库——Fade、Rotate、Scale、Resize、Spring、Bounce、Squash & Stretch——随时堆叠成你的构想。
- **物理缓动工坊。** 贝塞尔曲线与弹簧物理作为头等公民。为每个关键帧配上一条它应得的曲线。
- **空间动效。** 2D 变换叠加深度——把动效与 3D 变换结合，让物体像真实世界中那样运动。
- **屏与屏之间的编排。** 每一次转场都拥有独立完整的时间轴，逐元素调节时序。

### 专业动效图形，浏览器原生
专业动效图形引擎的深度，无需导出管线。每一项原语都可被 Agent 作为工具调用——总计超过 330 个工具，组织为 22 个原生系统。

**图层与合成原语**
- **动态模糊（Motion Blur）。** 基于速度的拖影，可调强度与快门角度——电影感 180° 默认，长拖影 360°，或锐利 45°。
- **空对象（Null Object）。** 隐形控制器图层，通过父子层级驱动整组——组织复杂绑定，不绘制一个像素。
- **路径修剪（Trim Path）。** 通过 `stroke-dasharray` / `stroke-dashoffset` 实现路径渐进揭示——笔画书写、Logo 线稿绘制、SVG 式揭示。
- **重复器（Repeater）。** 按变换偏移（x / y / 旋转 / 缩放）与不透明度衰减复制图层 N 次——从单一来源生成径向、线性或网格图案。
- **运动拖尾（Echo）。** 运动余像——N 个延迟副本，逐级衰减不透明度与可选缩放，构建追踪与拖尾效果。
- **时间重映射（Time Remap）。** 每层独立播放速率——2× 加速、0.5× 慢速、冻结于特定帧、或倒放。独立于全局时间轴。
- **图层效果（Layer Effects）。** 投影、内阴影、外发光、内发光、描边——全部由可叠加的 CSS `box-shadow` 实现，支持 hex+透明度颜色控制。
- **高级混合（Advanced Blending）。** 填充不透明度、按通道（R/G/B）混合、Knockout 组、`Blend If` 源范围遮罩。
- **预合成与折叠变换（Pre-compose & Collapse Transformations）。** 将选中图层嵌套为自包含合成；通过嵌套管线折叠变换。
- **Alpha 与转移模式。** Straight 与 Premultiplied Alpha；Stencil Alpha/Luma 与 Silhouette Alpha/Luma 轨道遮罩。
- **混合组（Blending Group）。** 在图层组内隔离混合，使合成在抵达父级堆栈前先行解析。

**3D 摄像机、灯光与光学系统**
- **3D 摄像机与灯光。** 带深度的摄像机变换；点光/聚光/环境光/平行光，锥角，带不透明度与模糊的投影。
- **景深（Depth of Field）。** 按摄像机控制焦距、光圈、模糊量；带 Bokeh 形状（六边形、八边形、刀片数）与焦点曲线控制的高级变体。
- **镜头畸变（Lens Distortion）。** 桶形 / 枕形 / 移除畸变，带强度与边缘权重参数。
- **色差（Chromatic Aberration）。** 按通道 RGB 分裂，径向或方向偏移——边缘处的色彩溢出。
- **暗角（Vignette）。** 边缘衰减变暗，带圆度、中点、羽化控制。
- **相机抖动（Camera Shake）。** 程序化手持抖动，带频率、振幅、种子——可跨渲染复现。
- **光流与匹配移动（Optical Flow & Match Move）。** 像素级运动向量用于运动估计；跟踪并应用运动，将图层锁定到源素材。
- **变形光晕（Anamorphic Flare）。** 水平镜头光晕条纹——电影感横向光晕。

**调色与色彩校正**
- **色阶、曲线、色彩平衡、色相/饱和度、自然饱和度、曝光、阴影/高光、可选颜色。** 完整的 SVG `feComponentTransfer` 链——每一阶段在浏览器中原生渲染。

**路径运算与形状创作**
- **形状布尔（Shape Boolean）。** 联合、减去、相交、排除——基于 SVG 路径运算，带强度混合。
- **多路径修剪、合并路径、偏移路径、收缩与膨胀、扭转、锯齿。** 非破坏性地创作与修改矢量形状。

**时间效果与节奏**
- **时间位移、高级回声、带过渡的序列、时间反转、定格帧、高级抽帧、时间重映射变速。** 变速曲线、过渡式序列化（交叉淡入/溶解/擦除/推移）与帧级节奏控制。

**文字动画系统**
- **范围选择器、文字抖动器、路径文字、垂直文字、字间距、行距、逐字符变换、文字动画器。** 按字符、词、行逐级动画——含沿任意路径排列。

**效果与滤镜库**
- **高斯 / 方向 / 径向模糊、锐化、波形扭曲、涟漪、膨胀、发光、马赛克、查找边缘、镜头光晕、四色渐变。** 全部渲染为 SVG 滤镜链——可堆叠、可动画。

**表达式引擎与动画助手**
- **表达式、循环表达式、序列图层、指数缩放、平滑关键帧、抖动关键帧、音频关键帧。** 活的表达式引擎，原生支持循环、缩放与音频驱动助手。

**绘画、克隆与遮罩系统**
- **画笔笔触、克隆图章、画笔设置、画笔显隐、擦除笔触、笔触动画器。** 压感笔触，带书写动画、克隆与画笔驱动遮罩显隐。
- **遮罩系统与轨道遮罩。** 每层多遮罩，支持 add/subtract/intersect/difference 模式；Alpha 与 Luma 轨道遮罩。

**数据驱动与音频反应**
- **数据绑定、数据源加载、数据驱动图表、音频驱动器、音频反应关键帧。** 加载 CSV/JSON 数据源，绑定到图表原语，按音频频段驱动动画。

**木偶与网格变形**
- **Puppet Pin 与 Mesh Warp。** 基于钉点的骨骼变形与网格扭曲——有机地弯曲、扭转、拉伸形状。

**粒子系统与渐变描边**
- **粒子系统。** 程序化粒子发射，带寿命、速度、重力、湍流。
- **渐变填充与渐变描边。** 填充与描边上的线性、径向、锥形渐变，带可动画的色标。

**运动跟踪与稳定**
- **跟踪点、摄像机跟踪、Warp 稳定器、应用跟踪、编辑运动路径、自动朝向。** 单点跟踪、3D 摄像机求解、带平滑度控制的稳定、路径跟随朝向。

**Rotoscoping 与抠像**
- **Roto Brush。** 基于种子点传播的自动遮罩，可调检测灵敏度、平滑度与帧范围。
- **Refine Edge。** 遮罩边缘羽化，带智能半径、柔度、去污染。
- **Color Key、Linear Color Key。** 色度键，带容差、边缘细化与羽化；多色匹配，覆盖 RGB/色相/饱和度/亮度通道。
- **Difference Matte。** 与参考帧差分，带阈值、前置模糊与反转。
- **Spill Suppression。** 移除绿/蓝/红色溢出，带亮度保留。
- **Matte Choker。** 多级 choke/spread，带迭代次数与灰阶控制。
- **Inner/Outer Key。** 通过内外路径隔离前景/背景，带羽化与阈值。

**转场库**
- **Block Dissolve、Card Wipe、Gradient Wipe、Iris Wipe、Linear Wipe、Radial Wipe、Venetian Blinds、CC Jaws Wipe。** 完整转场库——块网格、卡片翻转、亮度驱动擦除、多边形光圈、角度扫掠、时钟擦除、条纹揭示、多向锯齿。

**模拟与生成器**
- **CC Ball Action、CC Bubbles、CC Rainfall、CC Snowfall、CC Star Burst。** 程序化自然现象——球阵、上升气泡、下雨、飘雪、星爆，全量 count/speed/wind/wobble 控制。
- **Cell Pattern。** 14 种程序化图案（bubbles、crystals、static plates、tubular、spotted、cracked、steel、organic、stone rock、dried up、shatter、scales、turbulent、load bubbles）。
- **Audio Spectrum。** 沿路径的频谱可视化器，带厚度、双色、analog/digital 显示模式。
- **Radio Waves。** 扩散波环发射器，带频率、扩散、淡出与起止宽度。

**Stylize 效果**
- **Cartoon Effect。** 卡通着色器，带边缘厚度、着色阶数、轮廓颜色、边缘模式（inverted/drawn/lit/outline）。
- **Brush Strokes、Oil Paint、Watercolor。** 绘画式渲染——笔触尺寸、角度、密度；油画笔触尺寸、对比度、锐度；水彩边缘强度、湿度、纸张类型。
- **Emboss、Motion Tile、Scatter、Threshold。** 浮雕雕塑；带相位/偏移的拼贴重复；带颗粒的像素位移；带通道选择与可选半色调的二值化。

### 一次构建动效系统，处处复用
动效系统让整个团队快速在任何地方加入品牌一致的动效资产。

- **跨资产复用组件。** 把常用动作系统化——无需每次从零搭建。创建可在任意应用处生效的模式与组件。
- **Skill 管线。** 把任意动效打包为自包含、可被 AI 调用的 skill——同一原语跨项目、跨团队、跨 Agent 流转。

### 组件级精细调优
每一段动效由相互独立的组件构成。调一个，不动其余——单独调整某个元素的缓动、延迟或变换，而周围的编排保持原样。

### 一键复用到 HTML 与视频
选中的动效直接导出为生产可用的 HTML 或渲染好的视频片段。在你浏览器里运行的那个产物，就是你交付的那个产物。

- **多格式导出。** 把任意动画帧转为 MP4、GIF、WebM，或干净的 HTML/CSS——视目标而定。

### 动效直达代码
所有动效由真实、生产可用的代码支撑。

- **代码原生输出。** 检视整条动效时间轴，直接把动画代码复制为 CSS、JSON 或 React。
- **MCP 原生 Agent 上下文。** 通过 MCP 服务器把动画代码发给你的 Agent 编码工具。所有取值——缓动、时序、变换——端到端保留。

### 任意动效导出为 Skill 
这是 OpenMotion 的定义性理念。任意动效——你的、我们的、或重新混合的——都能被打包成一个 **skill**：自包含、可被 AI 调用的动效单元。放入你 Agent 的工具箱，它便成为跨项目、跨团队、跨流程可复用的能力。

### 天生为 Agent 设计
OpenMotion 从一开始就为被 AI Agent 驱动而构建，而非仅供人类操作。平台把动效暴露为头等可编程表面——既可经自然语言、也可经结构化调用进行查询、编辑与组合。

## 架构

OpenMotion 围绕五大奠基支柱组织——四大产品表面，加上让它们事务化、可检索、可编程的运行时基础设施：

| 支柱 | 职责 |
| --- | --- |
| **模板库** | 精选、持续增长的顶尖动效模板集合，开箱即活。 |
| **动效 Agent** | 对话式核心——把意图翻译为精确的参数与关键帧修改。 |
| **实时运行时** | 在真实网页与视频帧中运行动效；同一表面即交付表面。 |
| **Skill 管线** | 把任意动效打包为自包含、可被 AI 调用、可复用的单元。 |
| **运行时基础设施** | 检查点回滚、FTS5 全文检索、先规划后执行编排、MCP 原生 Agent 表面——让动效事务化、可检索、可编程的基座。 |

它们共同构成闭环：*选择 → 运行 → 精修 → 交付 → 复用*——每一步都被检查点守护、可被检索、可被任一 AI Agent 寻址。
